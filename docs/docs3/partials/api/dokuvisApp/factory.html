Factory components in dokuvisApp

Name Description
AuthenticationFactory

Service responsible for checking the user status.
From tutorial: http://code.tutsplus.com/tutorials/token-based-authentication-with-angularjs-nodejs--cms-22543
UserAuthFactory

Service responsible for contacting the login endpoint and validating the user (and logging out the user).
From tutorial: http://code.tutsplus.com/tutorials/token-based-authentication-with-angularjs-nodejs--cms-22543
TokenInterceptor

Service responsible for sending in the access token and the key along with each request to the server.
From tutorial: http://code.tutsplus.com/tutorials/token-based-authentication-with-angularjs-nodejs--cms-22543
ValidateResolve

Resolver: Check for valid token and user data.
AuthenticateResolve

Resolver: No access, if user is not logged in.
SkipResolve

Resolver: If the user is already logged in, take him to the home page.
ProjectResolve

Resolver: Check, if project exists and user is granted to access.
SubprojectResolve

Resolver: Check, if subproject exists.
Archive

$resource for archives.
Category

$resource for Category. A category can have multiple attributes (CategoryAttribute), which can be assigned to models.
API url: api/auth/project/:project/category/:id
CategoryAttribute

$resource for CategoryAttribute, that can be assigned to models. Usually in combination with Category.
API url: api/auth/project/:project/category/:cid/attribute/:aid
Comment

$resource for comments.
ConfirmService

Service providing a dialog with two buttons to confirm or abort an action.

// default values
alertDefaults = {
    backdrop: 'static',
    templateUrl: 'partials/alerts/confirmAlert.html',
    show: true
};
alertOptions = {
    abortButtonText: 'Abbrechen',
    actionButtonText: 'OK',
    headerText: 'Fortfahren?',
    bodyText: 'Sind Sie sicher?',
    type: 'warning'
};
Project

$resource for project related tasks.
ProjInfo

$resource for ProjInfo.
Source

$resource for sources/documents.
SpatializeInterface

Interface between imageViewer and webglView directives and spatializeModalCtrl to manage interaction across their different scopes.
Subproject

$http methods for subproject related tasks
Uploader

This factory creates an instance of FileUploader (angular-file-upload), so it can be used in multiple controllers. Authentication headers are set for uploading.
Utilities

This factory provides some helpful functions.
webglContext

This service initializes standard objects and materials for the 3D viewport and holds all mesh/geometry instances. So the 3D content is kept in the memory, even if the viewport (respectively webglView directive) gets destroyed, and doesn't need to be reloaded when the user returns to the 3D viewport. Materials etc. are also globally availlable via THREE.DokuVisTray.
